# Week 4 — Research Agent

**Folder:** `Week4Assignment/`

This assignment implements a research agent that combines web search, session memory, tool-call logging, and file reading into a single agent capable of answering multi-hop questions. The agent runs on OpenRouter using OpenAI-compatible tool calling and uses SerpApi for live web search.

### Files

| File | Description |
|------|--------------|
| `Week4_Agent.ipynb` | Main notebook containing the agent implementation: tool definitions, memory, logging hook, and demonstration |
| `sample_report.txt`, `sample_report.pdf` | Sample files generated by the notebook for the file reader demonstration (not tracked in version control) |
| `tool_calls.log` | Log file produced by the logging hook, recording every tool call with timestamps (not tracked in version control) |

### Setup

The notebook prompts for the following API keys at runtime if they are not already available as environment variables or in a local `.env` file:

- `OPENROUTER_API_KEY`
- `SERPAPI_API_KEY`

Both keys can be entered interactively when the notebook runs, or placed in a `.env` file inside `Week4Assignment/`.

### Run

```bash
cd Week4Assignment
jupyter notebook Week4_Agent.ipynb
```

Run all notebook cells from top to bottom. The first cell installs the required packages (`openai`, `requests`, `pypdf`, `fpdf2`, `python-dotenv`).

### Expected Output

- Structured tool-call logs with timestamps for every search, file read, and memory operation
- Agent responses to a multi-hop question, combining file content, remembered facts, and live web search results
- Final memory state showing facts stored during the session

### Implemented Features

- Web search using the SerpApi Google Search API
- Session memory with explicit fact storage and recall
- A logging hook that records every tool call
- A file reader supporting `.txt` and `.pdf` documents
- Multi-hop reasoning across file content, memory, and web search within a single agent
